# Proposal Patterns From User-Provided Examples

This reference distills reusable writing patterns from six local example documents:

- `重点研发项目-罗家俊-参与北大项目.pdf`
- `合同书--基于共格界面铅锡共混钙钛矿材料的X射线平板探测器研究.pdf`
- `横向-85万-合同扫描件-陈超项目.pdf`
- `湖北省基金一般面上项目华中科技大学申报书-陈超-最终版.pdf`
- `2023年重点研发计划项目-课题2任务书-唐江.pdf`
- `2020年-湖北省自然科学基金重点类项目计划任务书扫描版-2020-唐江.pdf`

Use these as structural and rhetorical patterns. Do not quote or claim details from the documents unless the user provides the relevant text.

## Common Proposal Logic

Strong applications usually follow this chain:

1. Field bottleneck:
   - Identify the limiting mechanism, not only an engineering inconvenience.
   - Example patterns: interfacial defects, carrier recombination, ion migration, phase instability, stress distribution, radiation response, X-ray conversion efficiency, detector dark current.
2. Scientific gap:
   - Explain why existing materials, structures, or process routes cannot answer the mechanism.
   - Avoid vague statements like "research is insufficient".
3. Core hypothesis:
   - Propose a mechanism-level hypothesis that can be experimentally verified.
4. Research contents:
   - Build 3-4 linked subtopics.
   - Each subtopic must produce evidence for the next.
5. Expected outcomes:
   - Connect mechanism understanding, performance indicators, papers/patents, and application scenario.

## National Key R&D / Task Book Pattern

Emphasis:
- Mission orientation.
- Task decomposition.
- Milestone assessment.
- Multi-party collaboration.
- Deliverable matrix.
- Engineering validation and reproducibility.

Recommended structure:
- Overall objective.
- Task-level objectives.
- Technical route.
- Annual or node-based milestones.
- Assessment indicators.
- Participant responsibility.
- Risk and coordination mechanism.

Writing style:
- Use clear target verbs: develop, establish, verify, demonstrate, integrate.
- Quantify acceptance indicators wherever possible.
- Distinguish project-level goals from subtask-level outputs.

## Natural Science Foundation Pattern

Emphasis:
- Scientific question.
- Originality.
- Mechanism.
- Hypothesis validation.
- Feasibility based on preliminary data.

Recommended structure:
- Background and scientific significance.
- Domestic and international progress.
- Core scientific questions.
- Research contents.
- Technical route.
- Innovation points.
- Annual plan.
- Feasibility and foundation.

Writing style:
- Convert engineering aims into scientific mechanisms.
- Use hypothesis-driven language: "拟阐明", "旨在揭示", "有望建立".
- Avoid overemphasizing product delivery at the expense of mechanism depth.

## Provincial Fund / General Program Pattern

Emphasis:
- Clear regional or disciplinary relevance.
- Feasible technical route.
- Moderate ambition.
- Strong preliminary foundation.
- Trackable annual outputs.

Recommended structure:
- Scientific problem and application background.
- Research target.
- Research content.
- Key innovation.
- Feasibility.
- Year-by-year plan.
- Expected outputs.

Writing style:
- Balance academic depth with feasibility.
- Use concise problem framing and avoid overly broad national-strategy language.

## Horizontal Contract / Enterprise Project Pattern

Emphasis:
- Application scenario.
- Engineering deliverable.
- Acceptance criteria.
- Schedule and payment-linked milestones when relevant.
- IP, confidentiality, and deliverable boundary.

Recommended structure:
- Project background.
- Technical objectives.
- Work content.
- Deliverables.
- Schedule.
- Acceptance criteria.
- Responsibilities and conditions.

Writing style:
- Keep scientific rationale concise.
- Prioritize measurable deliverables, test methods, and acceptance logic.
- Do not invent legal terms, payment terms, or IP ownership unless supplied.

## Perovskite / X-Ray Detector Project Pattern

Typical scientific themes:
- Coherent or semi-coherent interface regulation.
- Lead-tin mixed perovskite phase stability.
- Defect-state suppression.
- Carrier transport and recombination control.
- X-ray absorption, charge collection, dark current, sensitivity, detection limit, and imaging stability.

Useful mechanism framing:
- Interface lattice/strain coupling controls defect formation energy and carrier extraction.
- Component mixing affects phase stability, trap distribution, and radiation response.
- Microstructure regulation links crystallization kinetics to detector noise and sensitivity.
- Passivation or interface engineering shifts the balance among mobility, lifetime, and dark current.

Possible validation chain:
- Material structure: XRD, GIWAXS, SEM, AFM, TEM.
- Chemical/electronic state: XPS, UPS, Kelvin probe, FTIR.
- Carrier dynamics: TRPL, SCLC, impedance, transient photocurrent.
- Device performance: dark current, sensitivity, detection limit, linear dynamic range, response time, stability, imaging test.
- Mechanistic support: DFT, defect formation energy, finite-element field simulation, drift-diffusion model.

## Innovation Writing Pattern

Use this format:

- **创新点一：一句话提出机制或方法突破。**
  Explain how it differs from current methods. Explain what evidence will verify it.
- **创新点二：一句话提出结构、材料或过程突破。**
  Explain the expected effect on key bottlenecks.
- **创新点三：一句话提出 validation or application breakthrough.**
  Explain how the output enables device, platform, or engineering progress.

## Feasibility Writing Pattern

Argue feasibility from three dimensions:

1. Theoretical maturity:
   - Prior models, known physical/chemical principles, or literature basis.
2. Preliminary foundation:
   - Existing samples, data, SOPs, device results, patents, publications, or team experience.
3. Conditions:
   - Equipment, characterization platform, collaborator resources, cleanroom/process line, computation platform.

Never claim strong feasibility without evidence. If the user does not provide evidence, write "需补充前期数据支撑".
